# 퇴직연금 제도전환(DB→DC) 신청 확인 웹페이지 — 개발 기획서 (v3)

## 0. 배포/기술 전제
- 프런트엔드: GitHub Pages 정적 페이지 (순수 HTML/CSS/JS, 프레임워크 없이 가볍게)
- 백엔드 역할: Google Apps Script를 "웹앱"으로 배포해서 API처럼 사용
- 데이터 저장: Google Sheets (제출데이터 / 사번마스터 / 설정 3개 시트)
- 사용 환경: 사내망 아님. 개인 휴대폰에서 링크로 접속 → **모바일 반응형 최우선**
- 관리자: 1명 단독 관리, 비밀번호 기반 접근 (Apps Script 서버단에서 비교, 프런트엔드에 비번 절대 노출 금지)
- GitHub 저장소: Public

## 1. 배경
- 매년 2/1, 8/1자로 DB→DC 퇴직연금 제도전환 신청 접수 (연 2회)
- DB로 재전환은 재직기간 중 1회 가능 / DC 전환 후 전액 중도인출한 연도에 가능
- 당사 퇴직연금 간사기관: 교보생명

## 2. 신청자 페이지
### 2.1 상단 안내
- 제목: "DB → DC 퇴직연금 제도전환 신청자 세부내용 확인"
- 안내문구: "본 페이지는 DB→DC 퇴직연금 제도전환을 신청하신 분들의 세부 내용을 확인하기 위한 페이지입니다."
- 당사 퇴직연금 간사기관은 교보생명임을 상단 고정 노출
- 현재 접수회차(예: 2026-08) 자동 표시 (설정 시트 값 참조)

### 2.2 사번 확인(접근 통제) — 가장 먼저 진행
- 사번 + 성명 입력 → 사번마스터 시트와 대조
- 불일치 시: "사번 또는 성명을 다시 확인해주세요. 계속 문제가 있으면 인사담당자에게 문의해주세요" 표시, 이후 입력 차단
- 일치 시: 다음 단계로 진행, 사번마스터에 부서/직급 있으면 자동 채움(선택)

### 2.3 인적사항 입력
| 항목 | 형태 | 필수 |
|---|---|---|
| 사번 | 텍스트(검증됨) | Y |
| 성명 | 텍스트(검증됨) | Y |
| 부서 | 텍스트(본인 직접 입력) | Y |
| 직위 | 텍스트(본인 직접 입력) | Y |

### 2.4 DC 적용 희망 금융사 (라디오 버튼, 1개 선택)
1. 교보생명 (간사기관)
2. 삼성증권
3. 국민은행
4. 삼성화재
5. 농협은행
6. 신한은행
7. 미래에셋
8. 하나은행
9. 산업은행
10. 한국투자증권
11. 삼성생명
12. NH투자증권
13. 기타 (선택 시에만 텍스트 입력창 노출)

### 2.5 제출
- 입력 내용 요약(리뷰) 표시 → [제출하기]
- 필수 항목 미입력 시 제출 불가 + 하이라이트
- 제출 완료 시 "제출이 완료되었습니다" + 제출 일시 표시
- 동일 사번 재제출 시 최신 값으로 덮어쓰기(이전 제출시각은 별도 컬럼에 기록)

## 3. 관리자 페이지
- 별도 URL(예: /admin.html), 비밀번호 입력 화면
- 비밀번호는 설정 시트 셀에 저장, 비교는 Apps Script 서버단에서 처리
- 비번 분실 시: 본인 구글 계정으로 시트 열어 셀 값 수정 → 즉시 반영 (재배포 불필요)
- 목록 화면 컬럼: 접수회차 / 제출일시 / 사번 / 성명 / 부서 / 직급 / 희망금융사 / 상태
- 기능: 검색(사번/성명/부서), 필터(접수회차/금융사), 상세보기
- 엑셀 다운로드는 Google Sheets 자체 내보내기 기능으로 대체 (별도 개발 불필요)

## 4. Google Sheets 데이터 구조
### 시트1: 제출데이터
사번 / 성명 / 부서 / 직급 / 희망금융사 / 희망금융사_기타 / 접수회차(자동표시) / 제출일시(자동표시)

### 시트2: 사번마스터
사번 / 성명 / 부서 / 직급 (신청 대상자만 사전 입력, 검증 및 자동입력용)

### 시트3: 설정
관리자 비밀번호 / 현재 접수회차

## 5. 운영 프로세스 (접수 라이프사이클)
1. 접수 준비: 사번마스터 시트에 대상자 사번+성명 입력 (링크 발송 전 필수)
2. 링크 발송: GitHub Pages 링크를 카카오톡/문자/메일 등으로 개별 전달
3. 접수 진행: 신청자 사번 검증 후 제출 → 제출데이터 시트 누적, 관리자는 실시간 확인
4. 절차 종료: 사번마스터 시트 데이터 삭제 → 이후 해당 링크 접근 시 사번 검증 실패로 제출 불가 (자동 접근차단)
   - 제출데이터는 삭제하지 않고 그대로 보존

## 6. 보안 유의사항
- 관리자 비밀번호 비교는 반드시 Apps Script(서버)에서 처리. 프런트엔드/GitHub 코드에 비밀번호 원문 절대 포함 금지
- Google Sheet ID, Apps Script 배포 URL이 코드에 노출되어도 그 자체로 시트 편집권한을 주진 않지만, 시트 공유 설정은 '나만 보기' 유지
- 접근 통제의 핵심은 코드가 아니라 "사번마스터 시트 데이터 유무" (5장 참고)

## 7. 향후 확장 (2차)
- 중도인출 신청자가 사유(주택구입/요양/파산 등) 선택 시 필요서류 목록을 즉시 안내하는 별도 페이지
  - GitHub Pages + Google Sheets 구조 재사용 예정, 상세 요구사항은 추후 별도 기획
- 접수기간 외 접근 시 자동 마감 안내
- 제출 완료 시 신청자 본인 확인 메일/알림 발송(선택)
- DB 재전환(재직 중 1회) 신청 폼 확장

## 8. 오늘 개발 우선순위 (MVP)
1. Google Sheets 3종 + Apps Script 웹앱 기본 골격
2. 신청자 페이지: 사번 검증 → 인적사항 → 12개사 선택+기타 → 제출
3. 모바일 반응형 UI 점검 (실제 휴대폰 기준)
4. 관리자 페이지: 비밀번호 로그인 + 목록/검색/필터
5. GitHub Pages 배포 및 링크 테스트
